# Task 10 - Learning note

## What I learned
I learned that git add stages changes so they are ready to be included in a commit, but git commit is what actually saves those staged changes with a message.

## When it came up
It clicked when I kept writing changes in my file but they were not showing up in my commit until I realized I had not staged them properly with git add first.

## How I'd explain it to someone else
Think of git add like preparing items to ship in a box, and git commit like sealing the box and labeling it so it gets saved in your project history.
